How it works

From idea to reviewed code in five steps

Shannon sits between you and your Claude Code CLI. You define the agents, Shannon handles the rest.

1

Define Agents

Create agents with custom system prompts, model preferences (Opus, Sonnet, Haiku), and specific tool permissions. Each agent becomes a focused specialist.

2

Build Teams

Group agents into teams. Choose parallel, sequential, or custom execution. Use the visual canvas to draw complex dependency graphs.

3

Plan Tasks

Describe your goal in natural language. Shannon's AI planner breaks it into concrete tasks, maps dependencies, and assigns agents. Or write tasks manually.

4

Execute

Each task runs Claude Code CLI in its own isolated workspace copy of your project. Watch progress in real-time through the DAG visualization.

5

Review & Apply

Review every diff hunk by hunk. Accept, reject with feedback, or edit directly in Monaco. Nothing touches your project until you approve.

Features

Everything you need to orchestrate

A complete toolkit for managing Claude Code agents working in parallel on your codebase.

Custom Agents

Give each agent a specialized system prompt and choose their Claude model — Opus for complex reasoning, Sonnet for balanced speed, Haiku for simple tasks. Control exactly which tools they can access. A frontend agent, backend agent, test writer — each fine-tuned for their role.

Team Orchestration

Organize agents into teams with three execution strategies: parallel (all at once), sequential (chained one after another), or custom where you draw the dependency graph on a visual drag-and-drop canvas powered by React Flow.

AI Planning

Describe what you want to accomplish in natural language. Shannon uses Claude to decompose your goal into concrete, assignable tasks with proper dependencies. Includes a prompt improver that helps refine agent system prompts for better results.

Diff Review

The safety net. Accept or reject individual code hunks, approve or reject entire files, reject with written feedback that goes back to Claude for iteration, or modify workspace files directly in the built-in Monaco editor. Full control over every change.

Live Monitoring

Watch your session unfold in real-time through a directed acyclic graph that visualizes every task's status, dependencies, and progress. Stream Claude's output live as each agent works. Know exactly what's happening at every moment.

MCP Integration

Browse and install MCP servers from the Smithery registry, or configure your own. Attach servers to specific agents to extend their capabilities — web browsing, database access, API integrations, whatever your workflow needs.

Code review

Every change under your control

Nothing touches your project until you say so.

Shannon creates an isolated workspace copy for each task. When an agent finishes, you review every change through a detailed diff interface before anything gets applied to your real codebase.

  • Hunk-level review — accept or reject individual code chunks within a file
  • File-level control — approve or reject entire files at once
  • Feedback loop — reject with a reason, Shannon sends your notes back to Claude for iteration
  • Inline editing — modify workspace files directly in the built-in Monaco editor

Get started

Clone, build, orchestrate.

Shannon requires Claude Code CLI and Wails v2 installed on your machine.

$ git clone https://github.com/yourusername/shannon.git
$ cd shannon/frontend && npm install && cd ..
$ wails dev
